In this project, you will use Python, SQL, and Tableau to put together a complete analytics workflow, including:</p> <ul> <li>Data acquisition</li> <li>Data wrangling</li> <li>Data storage</li> <li>Data exploration and analysis</li> <li>Feature selection</li> <li>Machine learning model training</li> <li>Model evaluation</li> <li>Reporting and presentation of insights</li> </ul> <p>The final project is structured into 4 days in which you build up your deliverables progressively and iteratively. We have provided a list of data sets for you to choose from. However, you are encouraged to obtain your own data sets. Using the knowledge you have acquired and your experience working with data, you will come up with a plan for what you are going to do and then design the project around the data set you have chosen.</p> <p> <strong>You will be working individually for this project</strong>, but we'll be guiding you along the process and helping you as you go. You will be working on this project over the course of 4 days. The proposed progression for each day is listed below. However, if you finish a certain step ahead of time, you can proceed to the next step. In that case, you will conduct more iterations of your deliverables by for example deep cleanining your data, selecting better features, comparing different ML models, improving model predictions, and so on. Also, remember that data analysis is iterative. So from time to time you may need to step back to a previous phase and iterate. </p> <h3> <a id="user-content-day-1---brain-storming-and-data-preparation" class="anchor" href="#day-1---brain-storming-and-data-preparation" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Day 1 - Brain Storming and Data Preparation </h3> <ul> <li>Idea generation &amp; planning</li> <li>Data gathering &amp; cleaning</li> <li>Data storage</li> </ul> <h3> <a id="user-content-day-2---exploratory-data-analysis" class="anchor" href="#day-2---exploratory-data-analysis" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Day 2 - Exploratory Data Analysis </h3> <ul> <li>Data exploration</li> <li>Data visualization</li> <li>Data transformation</li> </ul> <h3> <a id="user-content-day-3---data-analysis-first-iteration" class="anchor" href="#day-3---data-analysis-first-iteration" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Day 3 - Data Analysis First Iteration </h3> <ul> <li>Feature selection</li> <li>Model training and evaluation</li> <li>Model evaluation</li> </ul> <h3> <a id="user-content-day-4---data-analysis-more-iterations" class="anchor" href="#day-4---data-analysis-more-iterations" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Day 4 - Data Analysis More Iterations </h3> <ul> <li>Iterations on your modeling</li> <li>Model evaluation and comparison</li> <li>Prepare for presentation</li> </ul> <hr> <h2> <a id="user-content-project-instructions" class="anchor" href="#project-instructions" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Project Instructions </h2> <h3> <a id="user-content-day-1" class="anchor" href="#day-1" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Day 1 </h3> <p>On the first day of your project, you will be planning your project, choosing a data set, downloading and cleaning it, and storing it in a MySQL database.</p> <p>To get started, explore and choose from one of the following data sets:</p> <ul> <li> <a href="https://www.kaggle.com/blastchar/telco-customer-churn" rel="nofollow">Telecom Customer Churn Data Set</a> </li> <li> <a href="https://www.kaggle.com/osmi/mental-health-in-tech-survey" rel="nofollow">Mental Health in Tech Survey Data Set</a> </li> <li> <a href="https://www.kaggle.com/usdot/flight-delays" rel="nofollow">Flight Delays and Cancellations Data Set</a> </li> <li> <a href="https://www.kaggle.com/nickhould/craft-cans" rel="nofollow">Craft Beers Data Set</a> </li> <li> <a href="https://www.kaggle.com/rhuebner/human-resources-data-set" rel="nofollow">Human Resources Data Set</a> </li> </ul> <p>These are all Kaggle data sets, so you can gain some context about them by looking at the field descriptions and the <em>Overview</em> tab for each data set. You can also get project ideas by looking at the different <em>Kernels</em> that people have created using the data sets and the approaches they have taken, but please <strong>do not plagiarize or copy someone else's work.</strong> Remember that the goal of this project is to demonstrate <em>your</em> skills, not someone else's. </p> <p>You are free to use other data sources from <a href="https://www.kaggle.com" rel="nofollow">Kaggle</a>, <a href="https://archive.ics.uci.edu/ml/datasets.html" rel="nofollow">UCI ML Repository</a>, <a href="https://www.data.gov/" rel="nofollow">data.gov</a>, public APIs, websites, etc. However you will need to spend more time to search and evaluate the quality and complexity of the data set you want to use. Discuss with your instructor as early as possible regarding your project ideas in order to avoid potential blockers. </p> <h4> <a id="user-content-project-ideas" class="anchor" href="#project-ideas" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Project Ideas </h4> <p>As you are evaluating the data sets and deciding which you would like to use for your project, think about the different types of information that could be extracted from each one and what problems you could potentially solve. You are not required to use machine learning but it is definitely a plus if you use it. It is totally fine if you are more of a business-focused person and use Tableau to extract business intelligence from the data. It is also fine if you are more of a traditional data analyst and primarily conduct statistical analysis on your data. But challenging your own limit is always beneficial.</p> <p>Below are some ideas for the data sets we selected for you which can help get you started with these data sets.</p> <h5> <a id="user-content-telecom-customer-churn" class="anchor" href="#telecom-customer-churn" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Telecom Customer Churn </h5> <p>This data set is great for practicing supervised machine learning. The most obvious supervised learning problem to model with this data set is attempting to predict churn (whether or not a customer is going to leave). Another supervised problem you can try to solve is predicting tenure, or how long a customer will end up staying with the company. There are also a lot of ways to slice, dice, and analyze this data set - for example, looking at how monthly charges increase or decrease depending on different types of contract, tenure, and services provided.</p> <h5> <a id="user-content-mental-health-in-tech" class="anchor" href="#mental-health-in-tech" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Mental Health in Tech </h5> <p>This data set is also good for supervised learning, where you would attempt to predict the whether the employee taking the survey has received treatment for mental health (treatment variable). You can also analyze how the other variables in the data set contribute to whether someone seeks treatment for mental health and explore what types of environments result in employees with the best mental heath states.</p> <h5> <a id="user-content-flight-delays-and-cancellations" class="anchor" href="#flight-delays-and-cancellations" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Flight Delays and Cancellations </h5> <p>This relational data set can help you practice your SQL joins as well as supervised learning as you attempt to predict which flights will be delayed and by how much time. There are also a variety of ways to explore this data set to determine which airlines are most efficient, what times of day there tends to be the most delays, which cities have more frequent delays than others, and how all this changes over time.</p> <h5> <a id="user-content-craft-beers" class="anchor" href="#craft-beers" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Craft Beers </h5> <p>This relational data set has fewer variables than the others, and that means that choosing it for your project will require you to apply some extra creativity. For example, you can create categorical variables out of the numeric abv and ibu variables to categorize a beer's strength and hopiness. You can also use string operations on the style field to extract additional keyword-based categories (e.g. IPAs, American, English, Wheat, etc.). For modeling, you can attempt to predict the style of the beer or the probability that it falls into a particular style. Alternatively, you can attempt to compute similarity between beers to determine which ones are most like which other ones.</p> <h5> <a id="user-content-human-resources" class="anchor" href="#human-resources" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Human Resources </h5> <p>This is another relational data set where you can practice your SQL joins as well as data wrangling, analysis, and machine learning. As far as applying machine learning to this data set, you can attempt to cluster employees together into similar groups, you can attempt to predict performance scores or pay rate, or you can attempt to predict whether an employee will be terminated within a certain amount of time. Additionally, you can also analyze the relationship between pay and performance by position, department, and manager as well as employee demographics.</p> <p>Once you have chosen a data set, you will need to download it. In order to do so, you will need to create a free Kaggle account if you don't already have one. Once you have downloaded the data, you should perform the steps below:</p> <ul> <li>Create a new MySQL database where your data will eventually be stored.</li> <li>Create a new Jupyter Notebook for your project.</li> <li>In the Jupyter Notebook, read the data files using Python.</li> <li>Perform any necessary data wrangling and cleaning using Python.</li> <li>Create a connection to your MySQL database using <code>pymysql</code> and <code>sqlalchemy</code> and write the clean version of the data to the database. </li> </ul> <h4> <a id="user-content-additional-ideas" class="anchor" href="#additional-ideas" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Additional Ideas </h4> <h5> <a id="user-content-analyze-your-own-facebook-data" class="anchor" href="#analyze-your-own-facebook-data" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Analyze Your Own Facebook Data </h5> <p>Do you know you can download your own Facebook data to your harddisk? You can use this data to do social network analysis or visualize your friendships.</p> <p> <strong>Warning: Downloading your Facebook data takes a lot of time.</strong> You need to first request your data in your Facebook account. Then Facebook will prepare your data for you to download as a ZIP file. If you request all your data (e.g. posts, photos, likes, friends, places, etc.), it takes at least a whole day for Facebook to get your data ready. But instead of requesting all your data, you can also request only the categories you want to analyze such as friends and places, which takes significantly less time. In either case, request your data as early as you can so that you can have the data before the project week begins. </p> <p>Besides, when you request your data from Facebook, choose if you would like the data to be in JSON or HTML format. It's a strategic choice depending on what topic you want to work on for your final project. If you request JSON, you will be able to focus on data visualization and analysis. If you request HTML, the data may be more suitable for natural language processing but you will need to spend time on parsing the HTML data.</p> <p>Resources:</p> <p> <a href="https://www.facebook.com/help/1701730696756992" rel="nofollow">Accessing &amp; Downloading Your Facebook Information</a> </p> <p> <a href="https://linkurio.us/blog/how-to-visualize-your-facebook-network/" rel="nofollow">How to Visualize Your Facebook Network</a> </p> <p> <a href="http://www.fmsasg.com/socialnetworkanalysis/facebook/" rel="nofollow">Facebook Social Network Analysis</a> </p> <h5> <a id="user-content-web-scraping" class="anchor" href="#web-scraping" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Web Scraping </h5> <p>The web is a free source of unlimited data. You can visualize virtually everything such as people, animals, countries, products, music, weather, Github, etc.</p> <p>Resources:</p> <p> <a href="https://blog.udacity.com/2015/01/15-data-visualizations-will-blow-mind.html" rel="nofollow">15 Data Visualizations That Will Blow Your Mind</a> </p> <p> <a href="https://carto.com/blog/eighty-data-visualizations-examples-using-location-data-maps/" rel="nofollow">80 Data Visualization Examples Using Location Data and Maps</a> </p> <h5> <a id="user-content-even-more" class="anchor" href="#even-more" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Even More </h5> <p> <a href="https://www.analyticsvidhya.com/blog/2018/05/24-ultimate-data-science-projects-to-boost-your-knowledge-and-skills/" rel="nofollow">24 Ultimate Data Science Projects To Boost Your Knowledge and Skills</a> </p> <p> <a href="https://www.quora.com/What-are-some-good-data-science-projects" rel="nofollow">What are some good data science projects?</a> </p> <h3> <a id="user-content-day-2" class="anchor" href="#day-2" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Day 2 </h3> <p>On the second day of your project, you will start with the clean data set you stored in your database on the previous day.
