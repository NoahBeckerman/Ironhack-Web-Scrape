class="anchor" href="#introduction" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Introduction</h2> <p>Business Intelligence is an application of data analysis that aids companies in making data driven decisions. Using Business Intelligence, technologies we can provide insights on the performance of a business in the past and the present as well as make predictions or recommendations for the future. A key difference between Business Intelligence and data analysis is that insights are meant to be actionable and geared towards helping a business make decisions.</p> <h2> <a id="user-content-key-concepts" class="anchor" href="#key-concepts" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Key Concepts</h2> <h3> <a id="user-content-data-data-warehouse-vs-unstructured-data" class="anchor" href="#data-data-warehouse-vs-unstructured-data" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Data: Data Warehouse vs. Unstructured Data</h3> <p>Businesses store vast amounts of data. Some for compliance reasons and other data specifically for deriving insights and historical reporting. The data can be stored in many forms including emails, invoices, and presentations but also in the form of structured tables.</p> <h4> <a id="user-content-data-warehouse-olap-and-oltp" class="anchor" href="#data-warehouse-olap-and-oltp" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Data Warehouse: OLAP and OLTP</h4> <p>Typically companies invest a great deal of resources in storing important data in relational databases. Data warehouses are typically a group of relational databases accessible using SQL. Many companies have two types of databases. <strong>OLAP</strong> (On-Line Analytical Processing) is a data store that is characterized by a relatively low volume of transactions. This database is typically used for complex queries and therefore the time it takes to write to the database can be long, but the read time needs to be short. On the other hand, <strong>OLTP</strong> (On-Line Transaction Processing) is a database that is used for a large volume of short on-line transactions. This type of database is used for fast query processing. Each database has advantages and disadvantages. By using both databases, we can optimize performance for both complex queries and a large volume of simple queries and insertions.</p> <h4> <a id="user-content-unstructured-data-data-lake" class="anchor" href="#unstructured-data-data-lake" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Unstructured Data: Data Lake</h4> <p>In the past, most of the data used to extract Business Intelligence was stored in relational databases. However, nowadays storage has become very cheap and technologies for distributed storage have been the standard for over a decade. Therefore, many organizations find it to be more cost effective to store their data unstructured and process the data at query time. A repository that contains both structured and unstructured data is called a data lake.</p> <h3> <a id="user-content-metrics" class="anchor" href="#metrics" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Metrics</h3> <p>A metric is a quantifiable measure that provides us with information about how well a company is doing at achieving its business objectives. Since companies can only improve the things they measure, it is a crucial part of an analyst's job to use the most appropriate metrics. Many companies like to use the term <strong>KPI</strong> (Key Performance Indicator) interchangeably with metrics. An example of a metric is the amount of money spent per sale or the percent of customers that buy a product out of all customers visiting the site.</p> <h3> <a id="user-content-actionable-insights" class="anchor" href="#actionable-insights" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Actionable Insights</h3> <p>Typically analysts try to create reports that contain insights that can produce an increase in revenue. While it is interesting to learn certain facts about the company and the products using data, we try to limit ourselves to recommendations that when acted on, will boost the bottom line.</p> <h2> <a id="user-content-summary" class="anchor" href="#summary" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Summary</h2> <p>In this lesson we learned about the key concepts of Business Analytics. We learned about different types of data stores as well as metrics and KPIs. In the following lessons, we will use Tableau to produce actionable insights.</p> </div> </div> </div> </div>
