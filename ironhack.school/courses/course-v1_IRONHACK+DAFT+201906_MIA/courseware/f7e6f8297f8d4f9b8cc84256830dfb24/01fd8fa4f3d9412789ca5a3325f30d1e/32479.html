id="user-content-introduction" class="anchor" href="#introduction" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Introduction </h2> <p>Programming is an essential skill for data analysts, data scientists, and especially data engineers. Although you are not a software engineer, writing good code is important for you because your colleagues will read your code and machines will execute your code. Your work is not worthy if no one can understand your code or the machine will break on your code.</p> <p>Then what code is <em>good code</em>? There are many criteria to judge the quality of the code but the bottom line is it needs to be simple and efficient. </p> <h2> <a id="user-content-simple-and-clean-code" class="anchor" href="#simple-and-clean-code" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Simple and Clean Code </h2> <h3> <a id="user-content-why" class="anchor" href="#why" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Why </h3> <p>Writing simple code allows you to be more productive and prevents your code from running into unnecessary errors. It also makes your code more readable, easier to be debugged, and easier to be updated. In product development, it's common sense that one should focus on a limited set of features and make each feature as efficient and usable as possible. Because the more features one attempts to achieve, the less effort will be put into improving each feature. In programming, it's the same as in product development.</p> <p>Programming has a human/social dimension. At the end of the day the machine is not the only consumer of your code. Your colleagues will read your code because coding is often team work. Your mentor/supervisor will read your code because they need to evaluate your work. You yourself will read your own code because you need to debug errors. If you are in the open-source community there will also be strangers reading your code because they need to build their codes on top of yours. The simplicity of code allows it to be easily understood by human programmers.</p> <h3> <a id="user-content-code-simplicity" class="anchor" href="#code-simplicity" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Code Simplicity </h3> <p>In order to write simple codes, you need to develop design thinking. There is a famous saying that <strong>everybody who writes software is a designer</strong>. You write codes to solve problems and to help people. What are the key issues to solve? What are the possible solutions? What is the right solution to use? These are the questions you need to ask yourself and it requires design skills to find the solution that works best. You will make autonomous decisions and you will need to put all your existing knowledge and all your past experience on the table to help you make the <em>right</em> decisions. You will also need to think from the shoes of your users to make sure your solution will indeed help them. </p> <p>The <a href="https://en.wikipedia.org/wiki/KISS_principle" rel="nofollow"> <strong>"keep it simple, stupid" (KISS) principle</strong> </a> states that your code should not attempt to achieve unnecessary, "smart" features. The more complicated your code is, the more error-prone and less maintainable it becomes. When you write a function to parse JSON data, for example, let it focus on parsing and nothing else. Don't introduce other functionalities in the same function such as removing special characters, converting letter cases, etc. If you really need to achieve extra goals, write additional functions. </p> <p>The KISS principle is related to the <a href="https://en.wikipedia.org/wiki/You_aren%27t_gonna_need_it" rel="nofollow"> <strong>"You Aren't Gonna Need It" (YAGNI)</strong> </a> and the <a href="https://en.wikipedia.org/wiki/Lean_software_development" rel="nofollow"> <strong>Lean Software Developoment</strong> </a>. These principles state that you shouldn't add features to your code unless you absolutely know they are needed. Taking the parsing JSON data example again, you shouldn't even write the functions to remove special characters or converting letter cases unless you have proof that it will be problematic if your code doesn't do that. Sometimes your data sources have already removed special characters so you don't need to do it in your code. Sometimes letter cases don't matter in the analysis. So try to finish writing the code as quickly as you can by achieving the minimal set of features. Then you come back to test your code with real data. If you find evidence that the features missing are causing probloms, you can then add those features. </p> <p>Also, an <em>easy</em> way does not necessarily mean a <em>good</em> way. For instance, you have written a piece of code to parse JSON and you find you just need to modify a few lines to make your code work for parsing other data formats such as XML and RSS. So you copy and paste the same piece of code several times and modify them in order to handle XML and RSS. <strong>BAD PRACTICE!</strong> You are breaking the coding principle called <a href="https://en.wikipedia.org/wiki/Don%27t_repeat_yourself" rel="nofollow"> <strong>"don't repeat yourself" (DRY)</strong> </a>. Copy-n-pasting is indeed easy for you to get things done. However, when people read your code it'll be a big waste of time. When you want to fix an error in your code you will also need to make the fix multiple times. The right approach here is to write a function with variables so that you can reuse the same piece of code for XML and RSS. </p> <h3> <a id="user-content-clean-code" class="anchor" href="#clean-code" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Clean Code </h3> <p>Clean code has a lot to overlap with simple code. It means you should keep your code as <strong>concise, readable, and maintainable</strong> as possible. If you can achieve your goal with one line of code, don't make it two lines. You should avoid making your code messy or cumbersome which the DRY principle previously mentioned can be of great help. You should also design your code with maintainable structure so that it can continue evolving as you build up your project. </p> <p>The <strong>readability</strong> of your code means your code should be self-explanatory about what it does. Some techniques to achieve good readability include: </p> <ul> <li> <p>Organizing your files. For example, if you have many files in your project you should create folders to hold different types of them such as spreadsheets, textual files, images, etc. Don't put all the files in one level as in a large project it'll be super difficult to find the files you want.</p> </li> <li> <p>Structuring your code. For example, place library imports at the beginning, define global variables next, then define functions, then write the rest of the code. Similarly, in a function, define the local variables first then write the code.</p> </li> <li> <p>Respect coding conventions (e.g. <a href="https://www.python.org/dev/peps/pep-0008/" rel="nofollow">Style Guide for Python Code</a>). Most importantly, follow naming conventions. For instance, in Python the following are the basic naming conventions: </p> <ul> <li>Use <em>CapWords</em> to declare class names. </li> <li>Use <em>words_connected_by_underscores</em> to declare variables and functions. </li> <li>Use <em>CAPITAL_LETTERS</em> to declare constants. </li> <li>Use descriptive names for all declarations. Examples: <code>parse_json</code> is better than <code>my_function</code> as a function name because it is self-explanatory on what this function does. And <code>raw_data</code> is better than <code>d</code> as a variable name because it clearly indicates what this variable is. </li> </ul> </li> </ul> <h2> <a id="user-content-code-efficiency" class="anchor" href="#code-efficiency" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Code Efficiency </h2> <p>Code efficiency is another dimension of your code that you will be increasingly concerned about as your role becomes more senior. In addition to making your code readable by humans your code should also be efficient to be executed on machines. Often times there are multiple ways to achieve the same goal and you should find out the way that gives your code the best performance. Improving your code efficiency requires fundamental knowledge about how the programming language is executed on machines and it's lifelong learning because new technologies and practices are constantly being improved/invented. In this course you won't need to worry too much about your code efficiency but it's great that you keep it in mind while you grow.</p> <p>In the context of this course, Python and SQL are where you can start developing your awareness for code efficiency. Below we provide some basic tips for you on how to write more effient code.</p> <h3> <a id="user-content-efficient-python-coding-tips" class="anchor" href="#efficient-python-coding-tips" class="octicon octicon-link"></span> </a>Introduction</h2> <p>Mapping, and filtering are important concepts in functional programming. These concepts come up in other distributed programming frameworks and exist in Python as well. In this lesson we will expand on the functional programming concepts we have learned in previous lessons and apply these concepts using mapping, reducing, and filtering.</p> <h2><a id="user-content-functional-programming-recap" class="anchor" href="#functional-programming-recap" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Functional Programming Recap</h2> <p>As we have learned in previous lessons, functional programming is a programming paradigm where code is written in such a way that avoids mutability or sharing state. Operations are performed by passing data through functions and storing the result in a new variable.</p> <h3><a id="user-content-immutability" class="anchor" href="#immutability" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Immutability</h3> <p>An immutable object is an object that cannot be changed after it is created. By sticking with functional programming, we ensure that no two processes will modify the same data. Instead, when each function modifies the data, it will then store the resulting data in a new variable. This results in cleaner, safer and easier to read code.</p> <h3><a id="user-content-transforming-state" class="anchor" href="#transforming-state" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Transforming State</h3> <p>When writing functionally, we can think of a function as a mapping from input to output. An example of shared state is a computer game where we have multiple characters moving on the screen. Moving one character might affect another character, even unintentionally. With functional programming, this is not a concern for us.</p> <h2><a id="user-content-mapping" class="anchor" href="#mapping" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Mapping</h2> <p>The goal of using the <code>map()</code> function is to apply a function to a sequence (like a list or a set). The <code>map()</code> function takes a function as an argument as well as a sequence and returns a sequence with the function applied to every element in the sequence. For example, let's create a function that divides a number by 2 and returns the result.</p> <pre><code>def half(x): return x / 2 </code> </pre> <p>Now that we have our function, let's apply it to a list of numbers.</p> <pre><code>l = [10, 12, 34, 23] map(half, l) &lt;map object at 0x000001CD4B9AA860&gt; </code> </pre> <p>The <code>map()</code> function creates a map object which is an iterable object. To create a new list, one option is to convert the iterable into a list.</p> <pre><code>list(map(half, l)) [5.0, 6.0, 17.0, 11.5] </code> </pre> <p>Similarly, we can cast the iterable into a set.</p> <h2><a id="user-content-filtering" class="anchor" href="#filtering" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Filtering</h2> <p>Like the <code>map()</code> function, the filter function takes a function and a sequence and returns an iterable. The goal of this function is to use the function we pass to it to remove elements from our sequence. Our function should return true for all the elements we want to keep and false for the ones we want to remove. For example, we can create a function that returns true if a number is odd and false if it is even. In fact, let's use a lambda expression for this task.</p> <pre><code>filter(lambda x: x % 2 == 1, l) &lt;filter object at 0x000001CD4B9AA940&gt; </code> </pre> <p>Again, this returns an iterable, so we will cast it to a list.</p> <pre><code>list(filter(lambda x: x % 2 == 1, l)) [23] </code> </pre> <h2><a id="user-content-reducing" class="anchor" href="#reducing" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Reducing</h2> <p>While the <code>map()</code> function applies the function to each element in the sequence, sometimes we might want to apply a function that will aggregate all elements in the sequence. There are built-in examples in Python for this like the <code>max()</code> function or the <code>sum()</code> function. The <code>reduce()</code> does exactly this. This function is not standard in Python and needs to be imported from the <code>functools</code> library. The <code>reduce</code> function starts from the beginning of the sequence and operates on two elements at a time. This is why the function passed to <code>reduce()</code> should always take two elements and return one.</p> <p>For example, if we would like to create a summation function using <code>reduce()</code>, we will sum two elements at a time.</p> <p>Let's write a lambda expression that will take two elements and sum them.</p> <pre><code>summation = lambda a, b: a + b</code> </pre> <pre>By passing this lambda expression to the <code>reduce()</code> function (along with a list <code>[10, 12, 34, 23]</code>), we will perform the following operations in this order:</pre> <pre><code>&gt; a = 10</code><br /><code>&gt; a = 12 + 10 = 22</code><br /><code>&gt; a = 22 + 34 = 56</code><br /><code>&gt; a = 56 + 23 = 79</code></pre> <p>Here is the completed snippet of code to find the sum:</p> <pre><code>from functools import reduce l = [10, 12, 34, 23] reduce(lambda a, b: a + b, l) 79</code> </pre> <p>Here is another example of <code>reduce()</code>. This time we will use <code>reduce()</code> to find the maximum of a list. We will do this by comparing two elements at a time and returning the largest of the two.</p> <pre><code>reduce(lambda a, b: a if a &gt; b else b, l) 34 </code> </pre> <h2><a id="user-content-functional-programming-in-pandas" class="anchor" href="#functional-programming-in-pandas" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Functional Programming in Pandas</h2> <p>In pandas, we can use the <code>apply()</code> function to apply a function to a dataset. We do not make a distinction between functions that are applied to every row or every column and aggregate functions. We can use the apply function for both types.</p> <p>Here is an example of a dataframe.</p> <pre><code>import numpy as np import pandas as pd df = pd.DataFrame(np.random.randn(4, 3), columns=['a', 'b', 'c']) df a b c 0 1.007118 -1.469496 -0.267296 1 -1.372287 1.451564 -0.333397 2 0.731125 -0.448180 1.779717 3 -0.903802 0.349003 -0.121670 </code> </pre> <p>We can use the <code>half()</code> function we defined earlier and apply it to every cell in the dataframe.</p> <pre><code>df.apply(half) a b c 0 0.503559 -0.734748 -0.133648 1 -0.686143 0.725782 -0.166698 2 0.365563 -0.224090 0.889859 3 -0.451901 0.174501 -0.060835 </code> </pre> <p>Furthermore, we can define an aggregate function that will return the range of a column (the difference between the maximum and the minimum values).</p> <pre><code>def range_func(x): return max(x) - min(x) </code> </pre> <p>When we apply the function to our dataframe, it will compute the range for each column by default.</p> <pre><code>df.apply(range_func) a 2.379404 b 2.921060 c 2.113114 </code> </pre> <h2><a id="user-content-summary" class="anchor" href="#summary" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Summary</h2> <p>In this lesson, we learned about an aspect of functional programming in Python. The <code>map()</code> function allows us to apply a function to an entire sequence. We use this function to map an input to an output in an immutable fashion. Similarly, the <code>filter()</code> function allows us to use a function to remove elements from a sequence that do not meet the condition specified by the function. The third function discussed in this lesson is the <code>reduce()</code> function. This function allows us to perform an aggregation on a sequence of elements. We perform this aggregation on two elements at a time. We have also learned how to apply functions to dataframes. We can use the same function both for applying a function to every cell and for aggregate functions. In upcoming lessons, we will be able to utilize these concepts to write code functionally.</p> </div> </div> </div> </div>

By default, Python processes each task you tell it to perform sequentially, which can take a substantial amount of time when either the number of tasks is large, the number of items on which you need to perform the task is large, or both.</p> <p>When the information you need to compute upon is large or the computation is complex, your program can take a long time to run. In cases like these, it is often useful to divide the work among the computing resources available on your machine. This is known as multiprocessing or parallelization, and it can save you significant amounts of time.</p> <p>Parallelization refers to the act of performing tasks in parallel. This is made possible by the existence of multiple CPUs or cores on a computer.</p> <p>Python has a <code>multiprocessing</code> library that allows you to spread the work your programs need to perform over the number of cores your machine has. In this lesson, we will learn how to parallelize our code using the multiprocessing library. Along the way, we will practice parallelizing via a variety of examples. Let's go ahead and import it so that it's ready to be used. </p> <pre> <code>import multiprocessing </code> </pre> <p>The multiprocessing library contains several tools for parallelizing your workflow, but the most commonly used is the <code>Pool.map</code> method. Just like the regular <code>map</code> function in Python, <code>Pool.map</code> applies a given function to an iterable such as a list or tuple. However, the main difference is that it does so in parallel. It is relatively straightforward to understand and should work for the majority of cases where you will need parallelization, so we will be focusing on this aspect of the multiprocessing library in this lesson. </p> <h2> <a id="user-content-getting-started" class="anchor" href="#getting-started" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Getting Started </h2> <p>The code examples in this lesson use the dataset <code>directories.zip</code> which you can download <a href="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/data/module-1/directories.zip" rel="nofollow">here</a>. After downloading the file, extract the content. You will have a folder called <code>directories</code> that contains sub-folders and files. </p> <h2> <a id="user-content-sequential-processing" class="anchor" href="#sequential-processing" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Sequential Processing </h2> <p>When you perform a series of tasks in Python, it will execute them sequentially, or one at a time. When a task is performed this way, a subsequent computation does not occur until the one before it is complete.</p> <p> <a Doing so would split the job we have to do across the number of cores our machine has (ex. 4 cores) and assign each core a task that it would perform at the same time as the other cores.</p> <p> <a href="https://camo.githubusercontent.com/15d01985b85db30ceed8908651872bc6277068c3/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f706172616c6c656c2d70726f63657373696e672e706e67" target="_blank" rel="nofollow"> <img src="https://camo.githubusercontent.com/15d01985b85db30ceed8908651872bc6277068c3/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f706172616c6c656c2d70726f63657373696e672e706e67" alt="Parallel Processing" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/parallel-processing.png" style="max-width:100%;"> </a> </p> <p>To do this in Python, all we would have to do is call the <code>Pool</code> class from the multiprocessing library and then simply <code>map</code> the function to the data. This would apply the function in a parallel fashion across however many cores your computer has at a time. Once that has happened, we call the <code>terminate</code> method to stop and shutdown the job and then the <code>join</code> method to prevent any "zombie" processes that may result from a child process continuing after a parent process has terminated. </p> <pre> <code>%%time pool = multiprocessing.Pool() result = pool.map(square, data) pool.terminate() pool.join() print(result) </code> </pre> <pre lang="text"> <code>[0, 1, 4, 9, 16, 25, 36, 49, 64, 81, 100, 121, 144, 169, 196, 225, 256, 289, 324, 361, 400, 441, 484, 529, 576, 625, 676, 729, 784, 841, 900, 961, 1024, 1089, 1156, 1225, 1296, 1369, 1444, 1521, 1600, 1681, 1764, 1849, 1936, 2025, 2116, 2209, 2304, 2401, 2500, 2601, 2704, 2809, 2916, 3025, 3136, 3249, 3364, 3481, 3600, 3721, 3844, 3969, 4096, 4225, 4356, 4489, 4624, 4761, 4900, 5041, 5184, 5329, 5476, 5625, 5776, 5929, 6084, 6241, 6400, 6561, 6724, 6889, 7056, 7225, 7396, 7569, 7744, 7921, 8100, 8281, 8464, 8649, 8836, 9025, 9216, 9409, 9604, 9801] CPU times: user 16.5 ms, sys: 19.9 ms, total: 36.4 ms Wall time: 40.3 ms </code> </pre> <p>We can see that this took a total of 36.4 milliseconds, which means it finished in 3.4% of the time it took to perform the task sequentially.</p> <h2> <a id="user-content-use-case-file-system-traversal" class="anchor" href="#use-case-file-system-traversal" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Use Case: File System Traversal </h2> <p>The example in the previous section was a simple one intended to show how easy it can be to parallelize operations using Python's multiprocessing library. In this section, we will take a look at a real-world use case where parallelization would be helpful and see how we can use the previous example as a guide in parallelizing our tasks.</p> <p>Suppose we had a directory with a hundred folders and hundreds of files in each folder. There are various file types contained within each folder, and we are specifically interested in knowing how many markdown files are in our directory and where they are located. These files can be identified by a .md file extension.</p> <p>To start, we will get a list of the directories using the <code>listdir</code> method from the <code>os</code> library. We are going to remove a hidden file called .DS_Store that was automatically created in the directory because it is not a folder and we want just a list of folders that we can search through. </p> <pre> <code>import os dirs = os.listdir('./directories') dirs.remove('.DS_Store') </code> </pre> <p>This list is an iterable, much like the list of numbers we used in the previous example.</p> <p>Next, we will define a function that will search through a given directory folder and extract a list of files that end with the .md file extension.</p> <pre> <code>def find_markdowns(dir): files = os.listdir(dir) markdowns = [dir +'/' + file for file in files if file.endswith('.md')] return markdowns </code> </pre> <p>This function can be applied to the iterable list just like our <code>square</code> function could be applied to the list of numbers in our earlier example. </p> <p>Let's apply this function to our directory list in a sequential fashion, time it the execution, and see how fast it can identify all the markdown files. To do this, we will apply the function inside a list comprehension to every directory in our directory list. This will produce a list of lists, which we will then flatten into a single list containing the location of all the markdown files as elements.</p> <pre> <code>%%time result = [find_markdowns(dir) for dir in dirs] flatten = [file for sublist in result for file in sublist] </code> </pre> <pre lang="text"> <code>CPU times: user 22.2 ms, sys: 49.3 ms, total: 71.4 ms Wall time: 70.8 ms </code> </pre> <p>We can see that the total time it took for this process to run sequentially was 71.4 milliseconds.</p> <p>Next, let's see if we can achieve any performance gains by applying the function to the iterable in parrallel. As before, we are going to call the <code>Pool</code> class and then map the function to the iterable. We are also going to flatten the results here just like we did for the sequential run. </p> <pre> <code>%%time pool = multiprocessing.Pool() result = pool.map(find_markdowns, dirs) pool.terminate() pool.join() flatten = [file for sublist in result for file in sublist] </code> </pre> <pre lang="text"> <code>CPU times: user 15.5 ms, sys: 23.1 ms, total: 38.6 ms Wall time: 133 ms </code> </pre> <p>This time, it only took 38.6 milliseconds to run, which is almost twice as fast as it took to run the process sequentially.</p> <h2> <a id="user-content-summary" class="anchor" href="#summary" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Summary </h2> <p>In this lesson, we introduced the concepts of sequential and parallel processing. We looked at a basic example of sequential processing and then parallel processing using the Python multiprocessing library and witnessed a significant improvement in performance using multiprocessing. We then used what we learned from the basic example and applied it to a real-world use case where we wanted to extract a certain type of file from a directory with a hundred folders and hundreds of files in each folder. We saw first-hand the performance gains that are possible with parallelization. As you work through the rest of the program, keep these concepts in mind and try to think about ways to implement multiprocessing when you have a time-consuming process that involves a large amount of data, very complex computations, or both.</p> </div> </div> </div> </div>
