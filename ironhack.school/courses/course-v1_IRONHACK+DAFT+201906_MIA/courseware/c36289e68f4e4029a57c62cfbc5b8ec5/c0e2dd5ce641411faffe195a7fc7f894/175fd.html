aria-hidden="true" class="octicon octicon-link"></span> </a>Lesson Goals </h2> <p>In this lesson you will learn about:</p> <ul> <li>Machine Learning pipelines.</li> <li>Why you need them.</li> <li>How to implement them in Scikit-learn.</li> </ul> <h2> <a id="user-content-introduction" class="anchor" href="#introduction" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Introduction </h2> <p>In fact you have seen machine learning pipelines in the previous lessons and labs though we haven't formally introuced the concept. What is a pipeline? In general, a pipeline is a complex process formed by subprocesses, so that the output of one subprocess is consumed as input by the following subprocess. Below is a simple example of using two Unix commands as a pipeline. The output of the first command is passed to the second command as its input by using the pipeline (|) connector:</p> <pre><code>$ cat /etc/mtab | grep sda5 </code> </pre> <p>The first command produces the content of the file /etc/mtab, which you do not see in the console because the content is consumed as input by the second command <code>grep</code>. The eventual output you see is what <code>grep</code> produces, a line containing the string <code>sda5</code>: </p> <pre><code>/dev/sda5 / ext4 rw,relatime,errors=remount-ro,data=ordered 0 0 </code> </pre> <p>Although the execution of this example may produce a different output in different computers, because the /etc/mtab file usually differs from one computer to another, the key point here is that the output of the first command (or subprocess) is automatically fed as input to the second command (or subprocess) in a transparent way. We only see the output produced by the last command (or subprocess) of the pipeline.</p> <p>Containing only two subprocesses, this small example provides an idea of how a minimalist pipeline works. A real Machine Learning pipeline may contain as many subprocesses as needed, as long as you relay the output of each subprocess to the next one.</p> <h2> <a id="user-content-machine-learning-pipelines" class="anchor" href="#machine-learning-pipelines" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Machine Learning Pipelines </h2> <p>We now transfer this concept to the domain of Machine Learning. In ML, the term pipeline is somewhat misleading because it sounds like ML is a one-way flow of data. On the contrary, however, ML pipelines are cyclic and iterative. Typically, one iteration of the ML pipeline consists the following subprocesses:</p> <ol> <li> <strong>Feature selection</strong> - select which feature set to be included in the ML model </li> <li> <strong>Training set formation</strong> - split the dataset into a training set and a testing set </li> <li> <strong>Training</strong> - use the training set to training the ML model </li> <li> <strong>Estimate test set</strong> - use the trained ML model to estimate the test set </li> <li> <strong>Evaluation</strong> - compare the estimated test set with the real test set </li> </ol> <p>Several iterations of ML pipelines are usually needed in order to improve/optimize you ML model. In each iteration, you can experiment different feature sets, different ML algorithms, or both. You save the state of your pipeline in each iteration. You keep performing as many iterations as you want but you should stop at a point when your further time investment is not rewarded by signficant performance gains of your ML model.</p> <p> <strong>All sounds good. But isn't that the same as conduting ML analysis with the same dataset but different algorithms???</strong> </p> <p>No they are not the same. You may have encountered the experience that when you perform the same ML analysis repeatedly, the outputs are different every time even if you are using exactly the same feature set and ML algorithm. This happens because when you split the train-test dataset each time, the randomization algorithm makes things different and your model performance scores become different. This makes it impossible to compare your model performance from iteration to iteration. Therefore, it's a good idea to take advantage of ML pipelines which makes sure you won't run into such troubles.</p> <h2> <a id="user-content-scikit-learn-pipelines" class="anchor" href="#scikit-learn-pipelines" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Scikit-learn Pipelines </h2> <p>The module <code>sklearn.pipeline</code> contains the class Pipeline that we can use to implement a Machine Learning pipeline. This class allows the implementation of a sequence of chained transformations that must end with an estimation. In other words, it creates a chain of transformers followed by an estimator. </p> <p>One advantage this implementation has is you do not need to build the pipeline every time it is going to be used: the whole pipeline can be serialized and saved to a file on its first use, and we can read the file an example below to show how this is done.</p> <h2> <a id="user-content-the-lymphography-application" class="anchor" href="#the-lymphography-application" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>The Lymphography Application </h2> <p>A lymphography is an X-ray image of a person's lymphatic system, including but not limited to the lymph nodes. Imagine you are provided with the lymphography data of 148 lymphographies obtained at a hospital from patients with metastases and malign lymphs, and you are asked to build a Machine Learning based system for diagnosing new patients based on the data of the 148 cases. How would you proceed? In this lesson we show you how to proceed, walking you through a process of 8 phases, that uses and exemplifies Machine Learning pipelines implemented in Scikit-learn. In your future Machine Learning projects, this 8 phase process can be reused and adapted, so that you do not have to start from zero, and you have an idea of how the whole process looks. So, let's get started!</p> <p>We will build a Machine Learning based application to diagnose metastases and malign lymph. For this purpose we will use the public <a href="https://archive.ics.uci.edu/ml/datasets/Lymphography" rel="nofollow">Lymphography dataset</a> available at the Machine Learning Repository at UCI (University of California at Irvine). We will break the process down into 8 phases: </p> <ol> <li>Load Dataset</li> <li>Slice Dataset</li> <li>Create Pipeline</li> <li>Train</li> <li>Evaluate</li> <li>Save the Pipeline</li> <li>Load the Pipeline</li> <li>Classify new instances</li> </ol> <p>We will explain what each stage does and its implementation, providing code snippets for each phase. At the end we will put all the snippets together and we will provide the whole Python program. Let's get started!</p> <h3> <a id="user-content-phase-1-load-dataset" class="anchor" href="#phase-1-load-dataset" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Phase 1: Load Dataset </h3> <p>For reading the CSV file into a Pandas dataframe, we use this function again (which we have already used in previous lessons):</p> <pre><code>import pandas.io def readCsvToDataFrame(path, row_with_column_names): theDataFrame= pandas.read_csv(path, header=row_with_column_names) return theDataFrame </code> </pre> <p>And we invoke like this:</p> <pre><code>dataset = readCsvToDataFrame(DATASETPATH, None) </code> </pre> <p>DATASETPATH is a constant that contains the path to the CSV file. It is usually a good idea to inspect the structure of the object just read and to preview a few lines. This is a handy function that you can use for the inspection:</p> <pre><code>def show_df_info(dataframe): # get the data type print(type(dataframe)) print("amount of entries is %s" % dataframe.size) print("dimensions= %i" % dataframe.ndim) print("shape is ", end="") print(dataframe.shape) print("axes: ", end="") print(dataframe.axes) print("data types of columns:") print(dataframe.dtypes) print("features: %s" % dataframe.columns) </code> </pre> <p>Then, we would invoke the function and preview a few lines:</p> <pre><code>show_df_info(dataset) print(dataset.head(5)) </code> </pre> <h3> <a id="user-content-phase-2-slice-the-dataset" class="anchor" href="#phase-2-slice-the-dataset" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Phase 2: Slice the Dataset </h3> <p>It is a good idea to encapsulate the details of the slicing that will vary from one software application to another, inside a function: that makes the rest of the code more reusable. In our example, we use this function:</p> <pre><code>def sliceDataFrame(df): # remove 2 instances with class label "normal find" df= df.drop(df[df[0]==1].index) # remove 4 instances with class label "fibrosis" df = df.drop(df[df[0] == 4].index) # iloc arguments: range of rows, range of columns # class 'pandas.core.series.Series' return df.iloc[:, 1:], df.iloc[:, 0] </code> </pre> <p>Note that in the slicing we have removed the instances of classes "normal find" and "fibrosis," because there are not enough of them to perform 5-fold cross validation. Then, using <code>iloc</code>, attributes have been selected, starting at column with index 1 and finishing at the last column. Also, using <code>iloc</code>, the class labels (target values) have been selected from the first column (index 0). </p> <p>Now we will invoke the function and then do some inspection and previewing of the dataframe afterwards:</p> <pre><code>training_instances, class_labels = sliceDataFrame(dataset) show_df_info(training_instances) # preview the data print(training_instances.head(5)) print() print(class_labels.head(5)) </code> </pre> <h3> <a id="user-content-phase-3-create-the-pipeline" class="anchor" href="#phase-3-create-the-pipeline" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Phase 3: Create the Pipeline </h3> <p>This is the code snippet that creates the Scikit-learn pipeline:</p> <pre><code>from sklearn import tree, pipeline, preprocessing from sklearn.feature_selection import SelectKBest from sklearn.feature_selection import chi2 cart_model = tree.DecisionTreeClassifier() pipe = pipeline.Pipeline(steps=[ ("feature_selection", SelectKBest(chi2, k=8)), ("scale", preprocessing.StandardScaler()), ("CART", cart_model)]) </code> </pre> <p>After the required imports, a CART model is created (as you may remember, this is something we did in a previous lesson), and then a pipeline is created with the following steps as argument:</p> <pre><code>steps=[ ("feature_selection", SelectKBest(chi2, k=8)), ("scale", preprocessing.StandardScaler()), ("CART", cart_model)] </code> </pre> <p>The first argument indicates the name of the step , in this case "feature_selection". The second argument is the invocation that implements the step, in this case: <code>SelectKBest(chi2, k=8</code>). <code>SelectKBest()</code> will evaluate each feature using chi-squared, and will return the best 8 features out of a total of 18 available features (this is called dimensionality reduction, which you can find more information about in our previous lesson on "Feature Extraction and Engineering"). </p> <p>The second step of the pipeline is "scale", and it scales the dataset resulting from the feature selection step, so that the domain of all features lies in the range [0, 1]. This is accomplished with this invocation:</p> <pre><code>preprocessing.StandardScaler() </code> </pre> <p>The third step in the pipeline "CART" and it applies the Machine Learning trained model to the scaled dataset output by the previous step.</p> <h3> <a id="user-content-phase-4-train" class="anchor" href="#phase-4-train" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Phase 4: Train </h3> <p>In the previous phase we created the pipeline, now we train it with a single invocation:</p> <pre><code>pipe.fit(training_instances, y=class_labels) </code> </pre> <p>Note that the fit() method is applied to all the steps in the pipeline, although there is a single invocation at the pipeline level.</p> <h3> <a id="user-content-phase-5-evaluation" class="anchor" href="#phase-5-evaluation" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Phase 5: Evaluation </h3> <p>We will evaluate the pipeline using the same method that we used in previous lessons: we run cross-validation and use as quality metric the average of the resulting scores. This is how you do it:</p> <pre><code># return value is array of scores scores = cross_val_score(pipe, training_instances, class_labels, cv=5) # use as quality metric the average CV score meanCvAccuracy = scores.mean() print("Mean CV accuracy= %f" % meanCvAccuracy) </code> </pre> <p>This is the output produced:</p> <pre><code>Mean CV accuracy= 0.760952 </code> </pre> <p>The interpretation of the output is that, based on the tests performed, the model is expected to provide the correct diagnose in 76% of new cases.</p> <h3> <a id="user-content-phase-6-save-the-pipeline" class="anchor" href="#phase-6-save-the-pipeline" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Phase 6: Save the Pipeline </h3> <p>Our pipeline is a chain of 3 steps, but we do not have to save them independently, we can save the whole pipeline with a single invocation:</p> <pre><code>joblib.dump(pipe, PIPELINEPATH, compress = 1) </code> </pre> <p>PIPELINE is a constant that contains the path to the file where the pipeline will be stored.</p> <h3> <a id="user-content-phase-7-load-the-pipeline" class="anchor" href="#phase-7-load-the-pipeline" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Phase 7: Load the Pipeline </h3> <p>Next time we need the pipeline, we do not have to build it again, we can reuse the pipeline we saved. It is read in the following code snippet:</p> <pre><code>pipeline_loaded = joblib.load(PIPELINEPATH) </code> </pre> <h3> <a id="user-content-phase-8-classify-new-instances" class="anchor" href="#phase-8-classify-new-instances" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Phase 8: Classify New Instances </h3> <p>We can use the pipeline we just loaded to classify new problem instances. In the following code, a new instance is created in variable vector, then it is processed by the pipeline, and the final classification result is printed:</p> <pre><code>import numpy # create new random problem instance vector = numpy.random.randint(0, 8, size=18) print(vector) result = pipeline_loaded.predict([vector,]) print("class label is %i" % result) </code> </pre> <p>This is the output to the console:</p> <pre><code>[0 1 0 5 4 7 1 3 4 2 2 2 0 2 7 1 1 2] class label is 3 </code> </pre> <h2> <a id="user-content-the-whole-program" class="anchor" href="#the-whole-program" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>The Whole Program </h2> <p>If we glue all the phases together, this is the resulting program:</p> <pre><code>import numpy from sklearn import datasets, utils from sklearn.model_selection import cross_val_score import pandas.io from sklearn import tree, pipeline, preprocessing from sklearn.feature_selection import SelectKBest from sklearn.feature_selection import chi2 from sklearn.externals import joblib # # DECLARATION PART # PIPELINEPATH= "ser_pipeline.pickle" DATASETPATH= "lymphography.data" # if there is no header row with the column/attribute names, use the constant None for row_with_column_names def readCsvToDataFrame(path, row_with_column_names): theDataFrame= pandas.read_csv(path, header=row_with_column_names) return theDataFrame def show_df_info(dataframe): # get the data type print(type(dataframe)) print("amount of entries is %s" % dataframe.size) print("dimensions= %i" % dataframe.ndim) print("shape is ", end="") print(dataframe.shape) print("axes: ", end="") print(dataframe.axes) print("data types of columns:") print(dataframe.dtypes) print("features: %s" % dataframe.columns) def sliceDataFrame(df): # remove 2 instances with class label "normal find" df= df.drop(df[df[0]==1].index) # remove 4 instances with class label "fibrosis" df = df.drop(df[df[0] == 4].index) # iloc arguments: range of rows, range of columns # class 'pandas.core.series.Series' return df.iloc[:, 1:], df.iloc[:, 0] # # PROGRAM BODY # ## PHASE 1: LOAD DATASET dataset= readCsvToDataFrame(DATASETPATH, None) show_df_info(dataset) print(dataset.head(5)) ## PHASE 2: SLICE DATASET training_instances, class_labels= sliceDataFrame(dataset) show_df_info(training_instances) # preview the data print(training_instances.head(5)) print() print(class_labels.head(5)) ## PHASE 3: CREATE PIPELINE cart_model= tree.DecisionTreeClassifier() pipe= pipeline.Pipeline(steps=[("feature_selection", SelectKBest(chi2, k=8)), ("scale", preprocessing.StandardScaler()), ("CART", cart_model)]) ## PHASE 4: TRAIN # fit all stages of the pipeline pipe.fit(training_instances, y=class_labels) ## PHASE 5: EVALUATE # return value is array of scores scores = cross_val_score(pipe, training_instances, class_labels, cv=5) # use as quality metric the average CV score meanCvAccuracy= scores.mean() print("Mean CV accuracy= %f" % meanCvAccuracy) ## PHASE 6: SAVE PIPELINE # the whole pipeline in one single file joblib.dump(pipe, PIPELINEPATH, compress = 1) ## PHASE 7: LOAD THE PIPELINE # read the file and deserialize the pipeline pipeline_loaded = joblib.load(PIPELINEPATH) ## PHASE 8: CLASSIFY NEW INSTANCES # create new random problem instance vector= numpy.random.randint(0, 8, size=18) print(vector) result= pipeline_loaded.predict([vector,]) print("class label is %i" % result) print("--- end of execution ---") </code> </pre> <h2> <a id="user-content-summary" class="anchor" href="#summary" aria-hidden="true"> <span aria-hidden="true" class="octicon octicon-link"></span> </a>Summary </h2> <p>In this lesson, you have learned what a Machine Learning pipeline is, its advantages, and the procedure to create one using Scikit-learn. You can reproduce the 8 phases shown here on your future Machine Learning projects, and adapt and reuse the code snippets. Now, you are ready to go!</p> </div> </div> </div> </div>
