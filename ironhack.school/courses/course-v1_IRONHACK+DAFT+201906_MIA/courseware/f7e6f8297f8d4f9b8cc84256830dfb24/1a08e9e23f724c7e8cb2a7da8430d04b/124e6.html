aria-hidden="true" class="octicon octicon-link"></span></a>Lesson Goals</h2> <p>In this lesson we will demonstrate how to create and resolve a merge conflict.</p> <h2> <a id="user-content-introduction" class="anchor" href="#introduction" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Introduction</h2> <p>When working collaboratively, sometimes we tend to step on each other's toes. What this means is that sometimes we and other developers happen to edit the same code at the same time. While we might try to keep our changes isolated with branching, this does not prevent editing the same lines of code by two different people.</p> <p>It is crucial that we learn how to resolve these conflicts as simply as possible. This is because the thing that scares away developers from Git the most is merge conflicts. We would like you to have strong Git Fu so we think you should tackle this problem head on. Let's start by setting up a merge conflict and then resolving it.</p> <h2> <a id="user-content-creating-a-merge-conflict" class="anchor" href="#creating-a-merge-conflict" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Creating a Merge Conflict</h2> <p>In order to create a merge conflict, we will create a Python file in PyCharm, create a new branch and change the code in the file. We will then change the code in the main branch and then proceed to merge the new branch with the main branch.</p> <h3> <a id="user-content-adding-a-file-to-the-repo" class="anchor" href="#adding-a-file-to-the-repo" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Adding a file to the repo</h3> <p>We open PyCharm and open the <code>data-labs</code> folder.</p> <p>In our <code>data-labs</code> repo, we will add a python file to this current module (module 1) and name the file <code>git-fu.py</code>. We right click the <code>module-1</code> folder and select Python file.</p> <p><a href="https://camo.githubusercontent.com/7638b326a118397f13dec93f2d19470e83220ba5/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6e65772d66696c652e706e67" target="_blank" rel="nofollow"><img src="https://camo.githubusercontent.com/7638b326a118397f13dec93f2d19470e83220ba5/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6e65772d66696c652e706e67" alt="python file" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/new-file.png" style="max-width:100%;"></a></p> <p>When PyCharm prompts you to add the file to Git, make sure to choose yes. You can also add it manually by typing this in the terminal:</p> <pre><code>git add . </code></pre> <p>We will add one line of code to the file for the sake of simplicity. The file contains only this line of code:</p> <pre><code>print("this is the original message") </code></pre> <p>Now we commit and push the repository using the following commands:</p> <pre><code>$ git commit -am "adding git-fu file" [conflict fdbdb3c] adding git-fu file 1 file changed, 24 insertions(+), 1 deletion(-) $ git push Enumerating objects: 18, done. Counting objects: 100% (17/17), done. Delta compression using up to 8 threads. Compressing objects: 100% (10/10), done. Writing objects: 100% (12/12), 31.53 KiB | 10.51 MiB/s, done. Total 12 (delta 3), reused 0 (delta 0) remote: Resolving deltas: 100% (3/3), completed with 2 local objects. To https://github.com/ironhack/data-labs.git 7f7f613..8b37c4f master -&gt; master </code></pre> <p>Now we will create a new branch and switch to this branch. We do this using the following command:</p> <pre><code>$ git checkout -b conflict-branch </code></pre> <p>We can also create a new branch in the git menu on the bottom right in PyCharm.</p> <p><a href="https://camo.githubusercontent.com/e3309fb3c89b2d2fe002c7d98c673d32e2445e7a/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6769742d6d656e752e706e67" target="_blank" rel="nofollow"><img src="https://camo.githubusercontent.com/e3309fb3c89b2d2fe002c7d98c673d32e2445e7a/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6769742d6d656e752e706e67" alt="git menu" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/git-menu.png" style="max-width:100%;"></a></p> <p>In the conflict branch we now alter the line of code in the <code>git-fu.py</code> file to this:</p> <pre><code>print("this is the altered message") </code></pre> <p>We now commit and push this change:</p> <pre><code>$ git commit -am "altering git-fu file" [conflict 775ed66] altering git-fu file 2 files changed, 35 insertions(+), 3 deletions(-) $ git push --set-upstream origin conflict-branch Enumerating objects: 11, done. Counting objects: 100% (11/11), done. Delta compression using up to 8 threads. Compressing objects: 100% (5/5), done. Writing objects: 100% (6/6), 1.08 KiB | 1.08 MiB/s, done. Total 6 (delta 2), reused 0 (delta 0) remote: Resolving deltas: 100% (2/2), completed with 2 local objects. To https://github.com/ironhack/data-labs.git * [new branch] conflict-branch -&gt; conflict-branch Branch 'conflict-branch' set up to track remote branch 'conflict-branch' from 'origin'. </code></pre> <h3> <a id="user-content-merging-the-branches" class="anchor" href="#merging-the-branches" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Merging the Branches</h3> <p>Now that we have set up the conflict, let's resolve it.</p> <p>We start by navigating to the master branch and merge the conflict branch into the master branch.</p> <pre><code>$ git checkout master $ git merge conflict-branch </code></pre> <p>Now we go back to the master branch and change the text again and commit and push it.</p> <p>The idea behind doing this is that the original text was determined in the master branch initially. In order to create a conflict, we must make sure both branches diverged from the original text.</p> <p>We alter the text to:</p> <pre><code>print("this is the master message") </code></pre> <p>We then commit and push this change to the master branch.</p> <p>Now we head back to the terminal and type:</p> <pre><code>$ git merge conflict-branch Auto-merging module-1/git-fu.py CONFLICT (content): Merge conflict in module-1/git-fu.py Automatic merge failed; fix conflicts and then commit the result. </code></pre> <p>Oh no! Our merge failed. Git wants us to fix this. We can head back to PyCharm and find that our <code>git-fu.py</code> file now looks like this:</p> <p><a href="https://camo.githubusercontent.com/2fc9105d8a7a21c911db9d5affdf628ef32ee4f5/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6769742d636f6e666c6963742e706e67" target="_blank" rel="nofollow"><img src="https://camo.githubusercontent.com/2fc9105d8a7a21c911db9d5affdf628ef32ee4f5/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6769742d636f6e666c6963742e706e67" alt="git conflict" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/git-conflict.png" style="max-width:100%;"></a></p> <p>You will also notice that our master branch is now called master|MERGING.</p> <p>In order to fix the conflict, we need to select the code we want to keep and erase the conflict boundaries.</p> <p>In this case, we want to keep the code in the master branch and so what we are left with is this:</p> <p><a href="https://camo.githubusercontent.com/9e50666627ea0317777723675e3f009f999ef92a/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6d61737465722d6d657267696e672e706e67" target="_blank" rel="nofollow"><img src="https://camo.githubusercontent.com/9e50666627ea0317777723675e3f009f999ef92a/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6d61737465722d6d657267696e672e706e67" alt="master merging" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/master-merging.png" style="max-width:100%;"></a></p> <p>We now commit the merged change. We should commit using an informative message and then push our code again.</p> <pre><code>$ git commit -am "resolving merge conflict" [master f360c5e] resolving merge conflict $ git push Enumerating objects: 14, done. Counting objects: 100% (14/14), done. Delta compression using up to 8 threads. Compressing objects: 100% (6/6), done. Writing objects: 100% (8/8), 721 bytes | 721.00 KiB/s, done. Total 8 (delta 4), reused 0 (delta 0) remote: Resolving deltas: 100% (4/4), completed with 3 local objects. To https://github.com/ironhack/data-labs.git 8b37c4f..f360c5e master -&gt; master </code></pre> <p>Now the conflict is resolved and the branches are merged.</p> <h3> <a id="user-content-aborting-the-merge" class="anchor" href="#aborting-the-merge" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Aborting the Merge</h3> <p>Perhaps you did not realize there was a conflict. Now there are hundreds of lines of code that you have to resolve. Maybe it would be better to use a different strategy. If you'd like to abort the merge, first we can check the status:</p> <pre><code>$ git status On branch master Your branch is up to date with 'origin/master'. You have unmerged paths. (fix conflicts and run "git commit") (use "git merge --abort" to abort the merge) </code></pre> <p>The <code>git status</code> command gives you the correct command to abort the merge. Now we will type it to go back to the pre-merged state</p> <pre><code>$ git merge --abort </code></pre> <p>As you can see, the branch name changed back from master|MERGING to master, so everything is back to normal.</p> <h3> <a id="user-content-resolving-the-conflict-using-ours-and-theirs" class="anchor" href="#resolving-the-conflict-using-ours-and-theirs" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Resolving the Conflict Using Ours and Theirs</h3> <p>Instead of resolving the conflict line by line, we can making a sweeping decision to accept a file from our branch as is (so from the master branch) or from their branch as is. To do this, we use the commands <code>ours</code> and <code>theirs</code>.</p> <p>In this example, before merging, we would navigate to the master branch. If we wanted to keep the master branch's version of <code>git-fu.py</code> we would type the following command before merging:</p> <pre><code>$ git checkout --ours -- ./module-1/git-fu.py </code></pre> <h3> <a id="user-content-conflicts-with-pull-requests" class="anchor" href="#conflicts-with-pull-requests" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Conflicts with Pull Requests</h3> <p>Pull requests are essentially a request to merge to branches on GitHub.</p> <p>We first push both branches and then create the pull request through the UI.</p> <p><a href="https://camo.githubusercontent.com/68070643110ab886e7bc139c8c171315112f79f3/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6372656174652d726571756573742e706e67" target="_blank" rel="nofollow"><img src="https://camo.githubusercontent.com/68070643110ab886e7bc139c8c171315112f79f3/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f6372656174652d726571756573742e706e67" alt="create pull request" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/create-request.png" style="max-width:100%;"></a></p> <p>As we can see in the screen shot above, the pull request will not automatically merge after approval. We still go ahead with creating the request, but we will have to resolve the conflicts after the request is created.</p> <p>After creating the request, we see the screen below. We click the Resolve conflicts button.</p> <p><a href="https://camo.githubusercontent.com/300e95383f40fb93ef9a280225c8207c9d3e3e51/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f7265736f6c76652d636f6e666c696374732e706e67" target="_blank" rel="nofollow"><img src="https://camo.githubusercontent.com/300e95383f40fb93ef9a280225c8207c9d3e3e51/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f7265736f6c76652d636f6e666c696374732e706e67" alt="resolve-conflict" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/resolve-conflicts.png" style="max-width:100%;"></a></p> <p>After clicking this button, we get taken to a UI that is very similar to what we see in PyCharm.</p> <p><a href="https://camo.githubusercontent.com/6d9fa89453acaa610b4f3920c9eadfbec3b7bc24/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f75692d6d657267652e706e67" target="_blank" rel="nofollow"><img src="https://camo.githubusercontent.com/6d9fa89453acaa610b4f3920c9eadfbec3b7bc24/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f75692d6d657267652e706e67" alt="ui merge" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/ui-merge.png" style="max-width:100%;"></a></p> <p>We resolve the conflict in the same way we resolved the conflict in PyCharm. The end result should be only the line of code that we want to keep. At this point, we are able to click the Resolve conflict button.</p> <p><a href="https://camo.githubusercontent.com/771d038ab586252e30e31468bc6b5ad1ee090e6f/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f75692d7265736f6c7665642e706e67" target="_blank" rel="nofollow"><img src="https://camo.githubusercontent.com/771d038ab586252e30e31468bc6b5ad1ee090e6f/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f75692d7265736f6c7665642e706e67" alt="ui resolve" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/ui-resolved.png" style="max-width:100%;"></a></p> <p>At this point, we go back to the main pull request page and see that the pull request is ready to merge.</p> <p><a href="https://camo.githubusercontent.com/0a0856cd17d993382ae792cea4e2ce76b0fdc32a/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f636f6e666c6963742d7265736f6c7665642e706e67" target="_blank" rel="nofollow"><img src="https://camo.githubusercontent.com/0a0856cd17d993382ae792cea4e2ce76b0fdc32a/68747470733a2f2f73332d65752d776573742d312e616d617a6f6e6177732e636f6d2f69682d6d6174657269616c732f75706c6f6164732f646174612d7374617469632f696d616765732f636f6e666c6963742d7265736f6c7665642e706e67" alt="merge resolved" data-canonical-src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/data-static/images/conflict-resolved.png" style="max-width:100%;"></a></p> <p>We are now able to safely merge the pull request</p> <h2> <a id="user-content-conclusion" class="anchor" href="#conclusion" aria-hidden="true"><span aria-hidden="true" class="octicon octicon-link"></span></a>Conclusion</h2> <p>In this lesson we learned how to resolve pull requests. We learned how to resolve them line by line and also force resolve them using <code>ours</code> or <code>theirs</code>. We also learned how to abort merges. Finally, we learned about resolving conflicts in pull requests on GitHub. Hopefully, you have reached a high level of Git Fu.</p> </div> </div> </div> </div>
